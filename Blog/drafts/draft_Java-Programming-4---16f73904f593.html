  1. Abstraction

(1) The Definition of Abstraction

Abstraction is something that exists only as an idea. For example, “Printing” is the idea of a program producing text as output. To think more about this concept, consider taking a philosophy course. Good abstractions simplify our view of the world, by unifying diverse real-world artifacts. In our programming context, abstractions are for delivering input to or receiving output from our programs.

(2) The Definition of Terminal

The terminal is an abstraction for providing input and output to a program

(3) The Definition of Command-Line Input

The command-line inputs are abstractions for providing arguments (strings) to a program. They are the strings you type after the program name and they are available as args[0], args[1], … at run time. Arguments are available when the program begins execution. Moreover, we need to call system conversion methods to convert the strings to other types of data.

(4) The Definition of Standard Output (Stream)

The standard output is an abstraction describing something having no limit (infinity). The standard output stream is an abstraction for an infinite output sequence.

  • Strings from System.out.println() are added to the end of the standard output stream. 
  • The standard output stream is sent to the terminal application by default.

(5) The Definition of Standard Input (Stream)

The standard input is an abstraction describing something having no limit. The standard input stream is an abstraction for an infinite input sequence.

  • Can provide new arguments while the program is executing.
  • No limit on the amount of data we can input to a program.
  • Conversion to primitive types is explicitly handled (stay tuned).
