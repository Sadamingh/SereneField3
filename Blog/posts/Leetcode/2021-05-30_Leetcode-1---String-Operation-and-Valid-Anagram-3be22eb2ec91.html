Leetcode 1 | String Operation and Valid Anagram

Series: Leetcode

Leetcode 1 | String Operation and Valid Anagram

242 Question:

Given two strings s and t, return true if t is an anagram of s, and false otherwise.

Constraints:

  • 1 <= s.length, t.length <= 5 * 104
  • s and t consist of lowercase English letters.

Example 1:

Input: s = "anagram", t = "nagaram"
Output: true

Example 2:

Input: s = "rat", t = "car"
Output: false
  1. C++ String Operations

(1) C++ String Data Structure

Strings in C++ are defined as an array of characters. The difference between a character array and a string is the string is terminated with a special NULL character \0.

(2) String Size

To get the size of a string, we have to basically access the string by the method size(). We will have O(1) complexity for getting the size of a string.

s.size()

(3) Find Substring

To search in the string is equivalent to search in an array. We need to use the method find() and the time complexity will be O(n). We will get a returned index back.

s.find("a")

(4) std::string::npos

Sometimes we search through a string for a character but nothing matches. Then the find function returns std::string::npos. You can find more information here.

if (s.find("a") == string::npos) 
    cout << "No \"a\" in the string" << endl;

(5) Delete Substring By Index

We can also delete the substring in a string by its index. For example, we can delete all the characters after index 5 by,

s.erase(5);

Or we can delete only the character on index 5 by,

s.erase(5,1);

Or we can delete characters with index 5 to 7 in the string by,

s.erase(5,3);

The time complexity of deletion is O(n).

Solution:

Idea #1. Sorting

The simplest idea is that we can simply sort all the characters in these two strings. Because they are all sorted, then we can simplest get the result by comparing these two sorted strings. However, the problem is that we can not control the sorting algorithm, and even if we can control the algorithms, they will not be efficient.

The example code is,

class Solution {
public:
    bool isAnagram(string s, string t) { 
        sort(s.begin(), s.end());
        sort(t.begin(), t.end());
        return s == t; 
    }
};

The result is,

Run time: 16 ms
Memory Usage: 7.3 MB

Idea 2. Find and Delete

The second idea to solve this problem is also simple. We can simply go through every letter in the first string s, find if it exists in the string t. If the letter exists, we will delete it from the string t and continue the loop. If the letter doesn’t exist, we will get npos back and this shows that s is not an anagram of t.

Note that there is also a case when the sizes of the two strings are different, so we have to make a judgment in the first place.

class Solution {
public:
    bool isAnagram(string s, string t) {
        int index;
        if (s.size() != t.size()) return false;
        for (int i = 0; i < s.size(); i++) {
            if ((index = t.find(s[i])) == string::npos) return false;
            t.erase(index,1);
        }
        return true;
    }
};

The result is,

Run time: 652 ms
Memory Usage: 7.3 MB

We can find out that because we have a time complexity of O(M*N) , then the run time for this idea will be much longer than the sorting one.

Idea 3. Hash Table

Based on the discussion above, we can know that the only way to improve our code is not to use any string operations like find or erase in the loop. And we can maintain the letter information of these two strings by using a 26-element array as our hash table. We can use this method because s and t consist of lowercase English letters, which makes the case simpler. Each of the 26 lowercase English letters corresponds to an index (or hash value that can be calculated simply by modifying its ASCII).

For each letter in s, it increases the counter by 1 while for each letter in t, it decreases the counter by 1. Finally, all the counters will be 0 if they two are anagrams of each other. So the example code should be,

class Solution {
public:
    bool isAnagram(string s, string t) {
        if (s.length() != t.length()) return false;
        int n = s.length();
        int counts[26] = {0};
        for (int i = 0; i < n; i++) { 
            counts[s[i] - 'a']++;
            counts[t[i] - 'a']--;
        }
        for (int i = 0; i < 26; i++)
            if (counts[i]) return false;
        return true;
    }
};

And the result will be,

Run time: 8 ms
Memory Usage: 7.3 MB

This one is perfect.
