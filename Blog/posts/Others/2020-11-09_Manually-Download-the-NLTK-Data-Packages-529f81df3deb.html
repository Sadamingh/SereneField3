Manually Download the NLTK DataÂ Packages

Suppose we are using the nltk package in python, for example,

from nltk.tokenize import word_tokenize
word_tokenize('He can can a can.')

then the output gives us the error like:

LookupError: 
**********************************************
  Resource punkt not found.
  Please use the NLTK Downloader to obtain the resource:

  >>> import nltk
  >>> nltk.download('punkt')
  
  For more information see: https://www.nltk.org/data.html

  Attempted to load tokenizers/punkt/PY3/english.pickle

  Searched in:
    - '/Users/apple/nltk_data'
    - '/Users/apple/opt/anaconda3/nltk_data'
    - '/Users/apple/opt/anaconda3/share/nltk_data'
    - '/Users/apple/opt/anaconda3/lib/nltk_data'
    - '/usr/share/nltk_data'
    - '/usr/local/share/nltk_data'
    - '/usr/lib/nltk_data'
    - '/usr/local/lib/nltk_data'
    - ''
*********************************************

then we would like to download the nltk_data packages (i.e. punkt). The normal way to download the data is to use the nltk downloader, by

import nltk
nltk.download('punkt')

However, sometimes we may have a connection problem if we are using a proxy,

[nltk_data] Error loading punkt: <urlopen error [Errno 61] Connection
[nltk_data]     refused>

Then we have to download the package manually.

Step 1. Go to GitHub repo, download the package we need and unzip this file.

For example, in this punkt case, we are going to download the zip file in this link. After we download it, we can then unzip it to get a folder named punkt.

Step 2. View the paths we searched in the LookupError. On my PC, the paths are,

Searched in:
    - '/Users/apple/nltk_data'
    - '/Users/apple/opt/anaconda3/nltk_data'
    - '/Users/apple/opt/anaconda3/share/nltk_data'
    - '/Users/apple/opt/anaconda3/lib/nltk_data'
    - '/usr/share/nltk_data'
    - '/usr/local/share/nltk_data'
    - '/usr/lib/nltk_data'
    - '/usr/local/lib/nltk_data'
    - ''

Step 3. Go to any one of these paths (if the folder does not exist, make a folder first)

$ mkdir /Users/apple/opt/anaconda3/nltk_data
$ cd /Users/apple/opt/anaconda3/nltk_data

Step 4. View the LookupError and find the loading path.

Attempted to load tokenizers/punkt/PY3/english.pickle

Step 5. Make a folder with the loading path directory name and go to this folder.

$ mkdir tokenizers
$ open ./

Step 6. Copy the package we have downloaded into this folder.

Then retry the python code,

from nltk.tokenize import word_tokenize
word_tokenize('He can can a can.')

We are supposed to have no error now,

['He', 'can', 'can', 'a', 'can', '.']
