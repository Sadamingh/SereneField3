一、关于Arduino的一切

Arduino是一种小型、廉价、开源的电子设备，通过易于使用的特性，它在业余电子社区变得非常流行。我们将会在后续的实验中使用Arduino微控制板进行设计。

Q：如何购买Arduino？

A：你有很多种选择去购买Arduino，由于Arduino是开源的，因此创造力是无穷的。你既可以选择Arduino官网上的版本，也可以选择其他开发商或者开发者的版本。

Q：我应该购买哪种型号的Arduino？

A：有数十种不同大小和功能的Arduino板，从技术上来讲，我们使用的并不是Arduino，而是Adafruit公司生产的与Arduino兼容的开发版Metro Mini。这款产品具有性能高，配件稳定，体积小，价格便宜的特点，官方售价是12.5美元，可以在图片下面的链接中找到。

https://www.adafruit.com/product/2590

如果上面的这个开发版在购买上存在难度，你也可以选择如下的开发版，它是Arduino UNO R3开发版，是Arduino公司自主生产的产品，虽然功能类似，但是体积较大，价格更贵。官网上的售价是23美元，可以在图片下面的链接中找到它。

https://store.arduino.cc/usa/arduino-uno-rev3

当然，可能大家需要的是一款相对更便宜的开发版，即使这将导致我们付出更多兼容性、调试和驱动上的努力，那么你可以考虑VISduino。它是一款中国生产的开发版，具有类似于Arduino UNO R3的特性，当然在某些地方可能会产生问题，但是相对于它的价格，看起来我们的努力是值得的。它的售价是4.9美元，非常不错。

Q：我如何安装Arduino编译器？

A：不论你是Mac用户、Windows用户、还是Linux用户，请从Ardunio官网上下载最新版的编译器（https://www.arduino.cc/en/Main/Software）。当然，你也可以使用在线编译器，但是没有任何保证它们一定能够工作！

Q：我如何测试IDE和我的开发版？

A：为了测试，我们需要将一个程序Blink写入开发版。

Step1. 选择开发版型号。工具 >> 开发版 >> Arduino UNO

Step2. 设置端口。

  • For Windows users, this should be COMx, where x is some number.
  • For Mac users, this will be devttyusbserial-xxxx, where xxxx may be some number.
  • For Linux users, this will be devttyUSB0 or possibly devttyACM0".

Step3. 打开Blink程序。文件 >> 示例 >> 01. Basic >> Blink

我们应该可以看到，在新弹出的窗口中有代码：

/*  Blink
  Turns an LED on for one second, then off for one second, repeatedly.
  Most Arduinos have an on-board LED you can control. On the UNO, MEGA and ZERO  it is attached to digital pin 13, on MKR1000 on pin 6. LED_BUILTIN is set to  the correct LED pin independent of which board is used.  If you want to know what pin the on-board LED is connected to on your Arduino  model, check the Technical Specs of your board at:  https://www.arduino.cc/en/Main/Products
  modified 8 May 2014  by Scott Fitzgerald  modified 2 Sep 2016  by Arturo Guadalupi  modified 8 Sep 2016  by Colby Newman
  This example code is in the public domain.
  http://www.arduino.cc/en/Tutorial/Blink
*/
// the setup function runs once when you press reset or power the board
void setup() {
  // initialize digital pin LED_BUILTIN as an output. 
     pinMode(LED_BUILTIN, OUTPUT);
}
// the loop function runs over and over again forever
void loop() {
    digitalWrite(LED_BUILTIN, HIGH);   // turn the LED on (HIGH is the voltage level)
    delay(1000);                       // wait for a second
    digitalWrite(LED_BUILTIN, LOW);    // turn the LED off by making the voltage LOW  
    delay(1000);                       // wait for a second
}

Step4. 点击 “上传” 按钮写入开发版

Step5. 你可以看到TX和RX灯不断闪烁，等待几秒钟后，你会看到Done uploading的字样，表示程序已经成功上传。

Q：我无法上传到开发版，怎么办？

A：如果报错为：problem uploading to board，请检查你下面的配置是否完成。

  • Check that you’ve set the board to Uno: Tools > Board > Arduino Uno.
  • Check that you’ve set the port correctly: Tools > Port. This can be confusing; you might have to try a few different ports to see which one matches. One way to determine which port it is, is to unplug the Arduino and see which one disappears, and plug it back in and see which one re-appears.

如果报错为stk500_getsync() download，原因是板上的针脚0和1用于对芯片进行编程，因此，如果将它们连接到5V或接地，或者如果它们负载很重（驱动LED），则编程信号将变得异常。

Q：我如何为开发版供电？

A：有四种方式可以为开发版供电。

  • 当您插入USB电缆时，Arduino将通过USB电缆供电。无需其他电源。在这种情况下，VIN应保持闲置状态。在5V引脚将由USB电缆供电，并且可以以此作为你的电路其余部分的5 V电源。
  • 您可以自己直接向5V引脚提供恒定的5 V（例如，使用实验室工作台上的电源）。在这种情况下，VIN应保持闲置状态。
  • 您可以向VIN引脚提供6 V至20 V之间的电压，虽然您可能不会这样做。板上有一个稳压器芯片，将使用它来产生5 V电源。该5V引脚将通过该供电，并且可以用它作为用于电路的其余部分的5 V源。
  • 您可以使用AA电池组为开发版供电。使用提供约4.5 V电压的AA电池组为Arduino开发板供电时，应将其连接到5V引脚。然后，请注意您的电路将由4.5 V而非5V供电。切勿在连接USB电缆的同时执行此操作，因为这会尝试将USB的5 V推入非充电式4.5 V电池。

Q：我还要注意什么？

A：您会注意到您的Arduino板上没有任何引脚。这不会阻止它运行，但是会使其很难连接到面包板上。因此，要使芯片与面包板配合使用，首先需要将引脚（包括在内）焊接到板上。它们为您提供了比您需要的更多的引脚，因此，请先使用剪钳切断多余的引脚，然后再将其焊接到板上。确保销钉垂直于电路板，否则将很难将其插入面包板。

如果您需要将Arduino从面包板中取出，请不要试图将一端拔出，然后将另一端拔出！这会将销钉弯曲到尴尬的位置。要么轻轻地将其拔出，要么使用IC提取器工具（在实验室中看起来像是多余的无用镊子的东西）。

连接USB电缆后，5V引脚将提供5 V电源，可用于为其余电路供电。不要忘记将所有需要接地的东西都连接到GND。

二、Arduino的引脚

D0 / RX / 0 接口：Receive接口，用于数据接收，数据输入（input）

D1 / TX / 0 接口：Transport接口，用于数据发送，数据输出（output）

D2～D13（2～13）接口：其他数据的输入输出接口

A0～A5接口：可以用作数据输入输出接口，也可以用作模拟输入接口，但是不能同时使用。

GND接口：接地口

RST接口：复位接口，如果将其链接到GND，那么将会重置Arduino。开发版上还有一个RST按钮，当它被按下时，会连接RST和GND，导致重置Arduino开发板。

VIN接口：可以用于提供电压，当输入6V～20V的电压时，会被稳压器转换为5V的输入电压。

5V接口：可以用于提供5V的稳定电压或者4.5V的AA电源电压。

3V接口：可以提供电压。但是实际上，3V接口在接入USB的时候是USB提供的3.3V电压。

三、I / O 引脚原理

在Arduino编程中，在使用一个接口之前，我们要用程序 pinMode() 指定它是一个输入接口 INPUT 还是一个输出接口 OUTPUT 。

输出OUTPUT引脚的原理图：输出引脚通过晶体管连接到VDD或接地，从而提供VDD或0V。你设置它使用函数 digitalwrite() 将状态设置为 HIGH（对于VDD来说）或 LOW（对于 0V 来说）。一个简化的原理示意图输出引脚如下所示。您可能会注意到，它看起来有点像CMOS反相器。

为什么我们需要一个类似于反相器的部分呢？这是因为软件输出的电压受到电阻和其他原因的影响，是低于VDD或者高于0V的。我们为了减小这种变化的影响，通过如图所示的电路可以将电压恢复到VDD或者0V的数值。

输入INPUT引脚原理图：输入引脚会像电压表一样读取引脚上的电压，如果电压接近VDD，则在软件中返回 HIGH （值为1），如果电压接近0V，则返回 LOW （值为0）。输入引脚可以使用 digitalread() 函数进行读取。使用 digitalread() 得到的返回值在不同设备不同软件中是不同的，尤其是电压的值介于二者之间的情况。

在我们的设备Merto mini中，引脚上的电压需要大于等于0.6VDD才被认为是 HIGH 。同样地，引脚上的电压要小于等于0.3VDD才被认为是 LOW 。、

上拉电阻Pull-up Resister：如果我们希望通过一个开关控制输入，当开关打开或者关闭的时候，input端会得到不同的电信号（开为 HIGH ，关为 LOW 或者开为 LOW ，关为 HIGH ）。那么如果我们只使用一个开关接地的话，我们就没有办法控制两种信号。比如下图所示，不论开关是否闭合，都只会输出0V也就是 LOW 。

所以我们就引入了设备电压，这样，如下图，在设备开关打开的时候，引脚感受到的就是设备电压，当设备开关关闭的时候，就相当于是短路，引脚感受到的是0V，从而实现不同信号的转换。

但是由于这种短路是不安全的，我们如下图所示接入一个电阻，这个电阻就是我们说的上拉电阻Pull-up Resister。

上面这个原理图也就等价于下面的图。我们可以看到，这个电阻是在Arduino开发板外面的。

但是，因为这个特性是如此得常用，以至于在Arduino内部实际上就有这样一块电阻来实现这个功能。下面的图表示了上拉电阻在Arduino内部的情况。是使用的时候，我们需要通过 pinMode(pin, INPUT_PULL) 程序指明某个接口为 INPUT_PULL 状态。这样做了之后，我们可以将这一接口连接到一个开关，从而通过一个外部开关的闭和开决定这个引脚输入的数值。

我们之所以能用 pinMode(pin, INPUT_PULL) 程序来指明是否使用上拉电阻，你可能已经猜到了原因，这是因为在上拉电阻的支路中存在一个用作开关的pMOS管。当调用模式为 INPUT_PULL 时，pull-up enable寄存器内的值被写为 HIGH ，我们对其取反后接到一个pMOS管的Gate上，和Source之间的电压达到阈值电压，从而将上拉电阻接入电路。具体原理图如下图所示。

四、引脚指令的快速参考

经过上面的讨论，对于引脚的输入，我们在 void setup() 有指令：

pinMode(pin , INPUT); // configures pin pin as input without pull -uppinMode(pin , INPUT_PULLUP); // configures pin pin as input with pull -up

对于引脚的输入，我们在 void loop() 有指令：

byte x = digitalRead(pin); // reads the value of pin pin

对于引脚的输出，我们在 void setup() 有指令：

pinMode(pin , OUTPUT); // configures pin pin as output

对于引脚的输出，我们在 void loop() 有指令：

digitalWrite (pin , HIGH); // sets the state of pin pin to HIGHdigitalWrite (pin , LOW); // sets the state of pin pin to LOW
